# Tunnel: Research

## Scope

Deep reading, web research, knowledge compounding, connecting ideas into living files.

## Boundaries

Do NOT execute system changes, install tools, or start projects. The output is knowledge, not infrastructure.

## Linked Files

- `03_Knowledge/` — write here
- `00_System/Nova_Timeline.md` — align research with sprint goals
- `01_Daily/<today>.md` — log findings

## Allowed Actions

| Action | Permission |
|---|---|
| Web search / fetch | Yes |
| Read and summarize | Yes |
| Create knowledge notes in 03_Knowledge/ | Yes |
| Update existing knowledge notes | Yes |
| Any system-level change | No |

## Rules

1. Every research output should be saved as a living file (frontmatter + markdown).
2. Link new knowledge to existing notes in the vault.
3. Tag findings with relevant tunnel and project references.
4. Flag anything that changes timeline priorities.
